### Confirmed, scheduled (not fixed today — see plan v3)

- **L1 — labels assume 1 row = 1 hour.** The hourly klines have 15 row-absent
  gaps (2–6h) per asset and zero NaN rows, so the documented "NaN-gap windows
  DROPPED" convention never fires and windows silently span more hours than
  bars around gaps. Fix = reindex the hourly grid (null rows) before labeling
  so the existing NaN discipline becomes operative. Changes labels near 15
  gaps → regenerate + re-verify + register; scheduled with the 4.4 close runs.
- **Rolling windows on gap-holed daily frames** (funding_z_30d etc.) span more
  calendar days than labeled around feed gaps — same family as L1, fix
  together (calendar densification before rolling features).
- **EWM warmup (min_periods=1)** — first ~60 rows per asset carry mis-scaled
  vol/momentum values into training. Fix with warmup masks when labels are
  regenerated (both changes alter the matrix → one registered matrix-v2 event).
- **`vol_realized` dead code** — `realized_from_intraday` never called;
  `vrp_daily` actually uses EWMA; block prefix lists a column that never
  exists. Wire it in or delete it at matrix-v2 time.
- **DuckDB backup** — `news.duckdb` first_seen stamps are unrecoverable;
  nightly copy to OneDrive goes into the ops hardening batch.
- **Farside browser-impersonation** (curl_cffi) if 403s persist after the
  retry+alert fix.

### The statistics verdict on published results

- **Meta-labeling 96h "+7.8pp"** (handoff headline): recomputed from the
  registry — the +7.8pp is the unweighted fold mean; event-weighted it is
  **+5.4pp with an across-fold SE of ~4.2pp (t≈1.85, one fold negative)**, and
  the biggest fold uplift (+20pp) sits on 13 gated events. A second threshold
  variant exists in the registry, so a selection effect sits on top. Honest
  status: *suggestive, not established*. Phase 5 return streams decide.
- **MCC 0.065/0.093 is in the honest published band** (Jaquart et al.: 52.9–
  54.1% accuracy for daily BTC classifiers). No red flag either way.
- **Lockbox residual scope**: `calibrate_k` still counts events over the full
  series (the k knob sees lockbox-period prices) and matrix building consumes
  2026 data causally — both are k-order effects accepted and documented here;
  the event/selection cutoff is what matters and is enforced.
- **Lockbox**: the 2026-01-01+ period was consumed by selection (finding 2).
  It is now sealed in code, but 2026-H1 must be reported as a *contaminated
  holdout* wherever it is evaluated. Recommendation (decision for Ian, see
  plan): keep the cutoff, re-run the key selection steps on pre-2026 data,
  treat 2026-H1 results as caveated, and let the pristine forward lockbox
  start today (2026-07-12+), untouched by construction.
